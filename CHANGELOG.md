# Changelog · video-script-creator

本文件记录 video-script-creator skill 的所有重要变更。版本遵循 [Semantic Versioning](https://semver.org/)。

## [3.0.0] - 2026-06-17

> **本次是真正的"v3"——v1→v2 内容字节级相同，v2→v3 才有实质升级。**
>
> 核心目标：把"个人作品"变成"可分发的产品"——补齐工程化短板，加 3 种调用模式 + fallback 风格 + 量化质量门。

### 重大变更（Breaking Changes）

- **拆分 SKILL.md**：490 行的单体文档拆为 `SKILL.md`（核心流程，≤300 行）+ `PRINCIPLES.md`（创意原则）+ `QUALITY.md`（质量门）。模型一次消化的内容从 490 行降到核心 200 行。
- **删除 `reference/` 单数僵尸目录**：v1/v2 里有 `reference/`（单数）和 `references/`（复数）并存，但 SKILL.md 只引用复数目录，单数目录里的内容无法被模型自动加载。v3 统一用 `references/`。
- **重命名 references 文件**：`01-...md` 编号命名 → 语义化命名（如 `style-xingqiu.md`）。插入新文件不再需要重编号。
- **frontmatter 规范化**：加 `version: 3.0.0` / `tags` / `mode` 字段，description 改为 YAML 折叠块。

### 新增（New Features）

- **3 种调用模式**：
  - 完整模式（默认）—— 6 阶段专业工作流
  - 快速模式 —— 局部创意产出（钩子 / 方案 / 金句）
  - 评估模式 —— 对已有方案做打分 / 优化
- **fallback 风格体系**（`style-fallback.md`）：支持 5 种备选风格——何同学 / 小约翰可汗 / 罗翔 / 影视飓风 / 学者派。星球所不再"唯一默认"。
- **案例解耦**（`case-template.md`）：把"福建微电网"案例拆为可复用模板 + 具体范例，避免方法论被单一案例绑死。
- **量化质量门**（`QUALITY.md`）：把"3 道递进有因果链"等主观标准，量化为可验证指标（金句密度 ≥1/2min、因果链关键词 ≥3 等）。
- **错误处理文档**（`errors-handling.md`）：明确什么场景应该拒绝 / 引导用户用别的工具。
- **assets/ 目录**：放可复制的 `proposal-template.md` 和 `script-template.md`，用户拿到即可用。
- **README + LICENSE**：让 zip 包可独立分发（v2 缺失这两个文件）。

### 改进（Improvements）

- **"团队 4 维度"改为内部检查表**：从"用户必问"改为"撰稿前自我反思"，减少用户疲劳（原 7-12 个问题 → 4 个）。
- **不适用场景清单**：在 SKILL.md 开头明确什么场景应该拒绝（如纯娱乐 / 直播带货 / 学术论文等）。
- **温度参数化**：从硬编码 0.7 改为按场景配置（0.3-0.85），并提供选择指南。
- **模型选择指引**：在 SKILL.md 加各阶段推荐模型。
- **去掉"商务项目人文化包装"的道德风险**（原则保留但加免责声明）：明确"产品是自然发现的，不是被推销的"是创作伦理底线。

### 内部重构（Refactoring）

- 给所有 reference 文件加 version / date / source 元信息块
- references 编号解耦
- 删除"团队 4 维度"作为用户必问，改为撰稿人自查表
- 福建案例单独成文，不再嵌入方法论

### 修复（Bug Fixes）

- 修复"v2 实质 = v1"的版本问题
- 修复"reference 单数目录"永远不会被加载的问题
- 修复"6 阶段硬编码"导致快速场景无法使用的问题
- 修复"星球所是唯一锚点"导致非地理类主题不适用的问题
