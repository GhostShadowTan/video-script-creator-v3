# 三连接规则详解（Three Connections Rule）

> 当用户要求连接 2 个及以上差异元素时（如福建的山 + 海、北京的历史 + 现代、人 + 技术），必须建立三个层次的连接，否则视频会显得"拼凑"。

---

## 1. 为什么需要"三连接"？

只有 1 个层次的连接（如"它们都用了同一项技术"）会让视频显得单薄。观众会问："这跟我有什么关系？"

通过 3 个层次的连接，可以：
- 让主题立得住（逻辑）；
- 让画面好看（物理）；
- 让人记得住（情感）。

---

## 2. 三连接的三个层次

### 2.1 技术 / 逻辑连接
- **是什么**：同一底层原理或技术 / 同一逻辑链条 / 同一规律
- **为什么需要**：让视频有"理性说服力"
- **范例**：
  - 福建的山与海：通过**季风 + 地形** 的逻辑连接（山抬升水汽形成降水，海送来水汽）
  - 北京的古今：通过**城市规划** 的逻辑连接（中轴线贯穿古今）
  - 人与技术：通过**算法** 的逻辑连接

### 2.2 物理 / 能量连接
- **是什么**：实际的流动 / 交换 / 物质循环
- **为什么需要**：让画面有"动感"和"过程感"
- **范例**：
  - 福建的山与海：通过**水循环** 连接（雨水从海蒸发，到山凝结，再流回海）
  - 北京的古今：通过**电力** 连接（旧城供电 + 新城用电的物理流动）
  - 人与技术：通过**电流** 连接（用户 ↔ 服务器 ↔ 用户）

### 2.3 人物 / 情感连接
- **是什么**：同一事物改变的真实人物的平行故事
- **为什么需要**：让观众有"代入感"
- **范例**：
  - 福建的山与海：山民 + 渔民 + 海岛居民，三种人共同的故事
  - 北京的古今：老北京人 + 新北京人，共同的命运
  - 人与技术：开发者 + 使用者，共同的体验

---

## 3. 应用案例：福建微电网

### 3.1 用户的"必须包含"元素
- 福建（地理特色）
- 微电网（科技产品）
- 海岛（场景）
- 商业合作（项目性质）

### 3.2 三连接的建立

**技术连接**：微电网 = 福建海岛 + 山地的统一答案
- 海岛需要独立供电 → 微电网；
- 山地村落难以架设长距离电网 → 微电网；
- 台风摧毁电网 → 微电网自愈能力。

**物理连接**：风 + 阳光 + 海流 → 电能
- 福建有 3000 公里海岸线 → 海上风电；
- 福建有亚热带季风气候 → 太阳能充裕；
- 福建有岛屿 + 山地 → 分布式能源；
- 微电网把"风光"变成"电"，把"电"送到"每户人家"。

**人物连接**：海岛教师 + 山村医生 + 渔村老人，共同的故事
- 海岛教师：因为微电网，学校的灯一直亮着，孩子可以上晚自习；
- 山村医生：因为微电网，卫生院的疫苗冷链不断，医生可以做手术；
- 渔村老人：因为微电网，海上归来的渔船有冷库，鱼可以卖得更远。

### 3.3 三连接带来的叙事深度
- **只有技术连接**：观众会觉得"这只是工程问题"。
- **加上物理连接**：观众会觉得"原来自然和科技在循环"。
- **加上人物连接**：观众会觉得"这改变的是真实的人的生活"。

## 4. 撰稿时的"三连接检查表"

| 元素 | 技术连接 | 物理连接 | 人物连接 |
|------|---------|---------|---------|
| 元素 1（地理 / 人文） | ... | ... | ... |
| 元素 2（科技 / 产品） | ... | ... | ... |
| 元素 3（场景 / 受众） | ... | ... | ... |

每行的 3 个格子必须都有内容。如果有任一格为空，说明这个元素的连接不完整，需要补。

---

## 5. 三连接的反模式（绝对避免）

| 反模式 | 现象 | 后果 |
|--------|------|------|
| **只有技术连接** | "X 和 Y 都用了 Z 技术" | 视频变成产品说明书 |
| **强行类比** | "X 像 Y，像 Z，像 W" | 没有真正的内在连接 |
| **虚假并列** | "X 和 Y 都是有趣的" | 没有因果链 |
| **缺失人物** | 整支视频没有人出场 | 观众无法代入 |

---

## 6. 实战范例：福建微电网方案 A「千岛之省」的三连接

### 6.1 技术连接
- 微电网的三大要素（源、网、荷）→ 福建海岛的共同解决方案；
- 海岛微电网抗台风、抗盐雾 → 福建的"必须条件"；
- 智能调度算法 → 跨越 1400 座岛屿的统一指挥。

### 6.2 物理连接
- 阳光 + 风 + 海流 → 微电网的"源"；
- 输电线路 → 微电网的"网"；
- 海岛居民用电 → 微电网的"荷"；
- 形成"福建海岛的能源小循环"。

### 6.3 人物连接
- 海岛教师：学校的灯；
- 海岛渔民：冷库的灯；
- 海岛医生：手术室的灯；
- 海岛老人：客厅的灯；
- 形成"1400 座岛的 1400 种守候"。

## 7. 撰稿自检清单

- [ ] 用户的每个"必须包含"元素都参与了 3 个连接？
- [ ] 技术连接让视频"理性立得住"？
- [ ] 物理连接让视频"画面好看"？
- [ ] 人物连接让视频"情感动人"？
- [ ] 至少有 2 个真实人物故事出场？
- [ ] 避免"虚假并列 / 强行类比"？
- [ ] 商务项目里，"主角"是土地与人，而不是产品？
