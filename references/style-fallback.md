# 备选风格体系（Style Fallback）

> **星球研究所是默认锚点，但支持 fallback 到 5 种其他风格。**
> 本文件给出每种 fallback 风格的"信号词 + 技法 + 反模式"。

---

## 1. 何时使用 Fallback

当用户说以下任一信号词时，从默认星球所切换到对应风格：

| 信号词 | 切换到 |
|--------|--------|
| "何同学" / "科技产品" / "工程师视角" / "理科生的浪漫" | Fallback 1：何同学 |
| "奇葩小国" / "硬核狠人" / "人物命运" / "神奇组织" | Fallback 2：小约翰可汗 |
| "段子" / "法理" / "张三" / "价值观" | Fallback 3：罗翔 |
| "工具评测" / "行业" / "团队纪录片" / "商业项目" | Fallback 4：影视飓风 Tim |
| "学者" / "院士" / "亲身经历" / "课堂" | Fallback 5：学者派（汪品先 / 韩茂莉） |

> **关键原则**：选 1-2 位 UP 主作为主参考，星球所风格作为克制的底色。
> 来源：[`creator-database.md` §终极提示](./creator-database.md)

## 2. Fallback 1：何同学（科技产品 / 工程师视角）

### 2.1 核心定位

> "用科技产品讲人和世界的关系。"
> 产品 → 技术 → 人 → 情感

### 2.2 风格特征

| 维度 | 何同学特征 |
|------|-----------|
| **钩子** | "做这个视频的起因是……" / 一个真实细节 |
| **主线** | 真实项目 / 真实问题 → 反复试错 → 解决 / 不解决 → 反思 |
| **金句** | "X 不是 Y 的工具，X 是 Y 的一部分。" |
| **节奏** | 慢节奏，停顿多，画面承担情绪 |
| **风险** | 制作成本高、依赖真实经历、节奏易拖沓 |

### 2.3 速用句式

- "我做这个视频，是因为……"
- "X 不是 Y 的工具，X 是 Y 的一部分。"
- "技术参数背后，是人和人的关系。"
- "把 N 个 M 放在一起，会发生什么？"
- "我用了 X 天，做了一件事。"

### 2.4 适合场景

- 科技产品评测（手机 / 电脑 / 智能家居）
- 工程师个人项目
- 科技公司 / 实验室的"幕后故事"
- 商务项目（特别是产品露出）

### 2.5 反模式

- 不要"硬科普"（不写论文体）
- 不要"先讲技术后讲人"（何同学是"先讲人后讲技术"）
- 不要"完美产品介绍"（必须有"翻车"或"反思"）
- 不要"参数堆砌"（参数背后要有人）

### 2.6 星球所底色如何叠加

- 保留"克制美学"（不写"令人震撼"）
- 保留"具体名词 + 精确数字"
- 去掉"宏大时间尺度"（何同学不做"6500万年"）
- 加上"真实经历 + 真实项目"

## 3. Fallback 2：小约翰可汗（人物命运主线）

### 3.1 核心定位

> "奇葩小国 + 硬核狠人 + 神奇组织"
> 人物命运 + 国家兴衰 + 冷知识

### 3.2 风格特征

| 维度 | 小约翰可汗特征 |
|------|--------------|
| **钩子** | 人物悬念 + 反差前置（"他做了 X，结果 Y，最后 Z"） |
| **主线** | 一个人物 / 一个组织 / 一个国家的"命运转折" |
| **金句** | "他不是 X，他是 Y。" |
| **节奏** | 中等节奏，每 30-60 秒有"小高潮" |
| **风险** | 资料准确性、人物评价客观性、可能"猎奇" |

### 3.3 速用句式

- "今天我们要聊一位……他做了……结果……"
- "他不是 X，他是 Y。"
- "看似是 X 的问题，其实是 Y。"
- "如果你以为 X，那是因为你不知道 Y。"

### 3.4 适合场景

- 历史人物故事
- 当代硬核人物（科学家 / 工程师 / 探险家）
- 公司 / 组织兴衰史
- "小众但硬核"主题

### 3.5 反模式

- 不要"猎奇"（不是讲八卦，是讲命运）
- 不要"道德审判"（小约翰不预设善恶）
- 不要"资料不全"（小约翰会做大量调研）
- 不要"英雄叙事"（小约翰更关注"复杂"）

### 3.6 星球所底色如何叠加

- 保留"事实优先"
- 保留"三连接"（技术 + 物理 + 人物中的人物连接尤为重要）
- 去掉"宏大地理视角"（小约翰不写"6500万年"）
- 加上"人物行动 + 命运转折"

## 4. Fallback 3：罗翔（段子 + 法理 / 价值观）

### 4.1 核心定位

> "用段子讲严肃法理"
> 案例 → 法理 → 人性 → 反思

### 4.2 风格特征

| 维度 | 罗翔特征 |
|------|---------|
| **钩子** | "同学们好啊" + 一个通俗案例 |
| **主线** | 案例 → 法理 → 人性 → 反思 |
| **金句** | "法律是对人最低的道德要求。" |
| **节奏** | 段子密，但每段段子后都有"法理升华" |
| **风险** | 段子过密易消解严肃性、案例选择不当易引争议 |

### 4.3 速用句式

- "今天张三又来了。"
- "如果 X，那 Y 是不是 Z？"
- "看似是 X 的问题，其实是 Y。"
- "法律是对人最低的道德要求。"

### 4.4 适合场景

- 价值观议题（科技伦理 / 社会现象）
- 案例分析（"如果 X 发生，会怎样"）
- 法理 / 制度解读
- 反常识社会议题

### 4.5 反模式

- 不要"段子堆砌"（每个段子后必须有"法理升华"）
- 不要"道德绑架"（罗翔从不预设善恶）
- 不要"案例失实"（法理必须严谨）
- 不要"长篇说教"（用案例讲，不是用道理讲）

### 4.6 星球所底色如何叠加

- 保留"事实优先 + 数据来源"
- 保留"克制"（不写"气死了"）
- 去掉"地理 + 时间尺度"（罗翔不写"6500万年"）
- 加上"通俗案例 + 法理升华"

---

## 5. Fallback 4：影视飓风 Tim（工具 × 故事 × 行业）

### 5.1 核心定位

> "工具评测升级为行业纪录片"
> 工具 → 团队 → 行业 → 价值

### 5.2 风格特征

| 维度 | 影视飓风 Tim 特征 |
|------|-----------------|
| **钩子** | "为什么我们要做这个？" / 一个行业痛点 |
| **主线** | 工具 → 团队 → 行业 → 价值 |
| **金句** | "工具的价值，是让人可以更专注于创造。" |
| **节奏** | 工具评测 + 行业洞察结合，节奏快 |
| **风险** | 商业味过重、行业洞察易主观 |

### 5.3 速用句式

- "为什么 X 行业 / Y 用户 需要这个工具？"
- "技术参数背后，是 N 个真实的 Z。"
- "做这件事最难的部分，不是 Z，是 W。"
- "今天我们不只看产品，我们看一个行业。"

### 5.4 适合场景

- 工具评测（相机 / 镜头 / 软件 / 设备）
- 行业纪录片（拍摄行业 / 设计行业 / 制造业）
- 商务项目（特别是企业品牌）
- "幕后制作"主题

### 5.5 反模式

- 不要"纯产品评测"（必须有行业洞察）
- 不要"软广"（即使是商务项目也不能"念稿"）
- 不要"技术参数堆砌"（参数背后要说"为什么需要"）
- 不要"个人英雄主义"（影视飓风强调团队）

### 5.6 星球所底色如何叠加

- 保留"事实 + 数据"
- 保留"克制美学"
- 去掉"宏大叙事"（影视飓风不写"6500万年"）
- 加上"行业洞察 + 团队视角"

## 6. Fallback 5：学者派（汪品先 / 韩茂莉）

### 6.1 核心定位

> "一线科研见闻 + 学者人格魅力"
> 学术问题 → 实地考察 → 回到课堂

### 6.2 风格特征

| 维度 | 学者派特征 |
|------|----------|
| **钩子** | 亲身经历 + 数据来源 |
| **主线** | 学术问题 → 实地考察 → 回到课堂 |
| **金句** | "我 82 岁那年，做了一件事……" |
| **节奏** | 慢节奏，学术性强，但有人格魅力平衡 |
| **风险** | 学术化、节奏慢、不适合短视频 |

### 6.3 速用句式

- "我做了 X 年研究，发现……"
- "科学家花了 X 年时间，才搞清楚 Y。"
- "这是一个被误解了 X 年的问题。"
- "如果我们回到 X 年前……"

### 6.4 适合场景

- 自然科学（地质 / 海洋 / 生态 / 天文）
- 历史人文（学者讲座 / 课堂）
- 工程前沿（院士视角）
- 学术科普（不是论文，是"学者说人话"）

### 6.5 反模式

- 不要"论文体"（不写摘要 / 方法 / 结论）
- 不要"数据轰炸"（每个数据要带"人话翻译"）
- 不要"居高临下"（学者不是"教育"观众，是"分享"）
- 不要"远离现场"（必须有"实地"或"亲历"细节）

### 6.6 星球所底色如何叠加

- 保留"事实 + 数据来源"
- 保留"克制"
- 保留"时间尺度"（学者派也用"6500万年"）
- 加上"亲身经历 + 人格魅力"

## 7. 风格混搭决策树

```
用户给的主题偏向...
│
├── 地理 / 城市 / 山河 → 星球所（默认）
│
├── 科技产品 → 何同学
│   ├── 工程纪录片 → 影视飓风 Tim
│   └── 商业项目 → 何同学 + 星球所底色
│
├── 人物 / 命运 → 小约翰可汗
│   ├── 科学家 → 学者派
│   └── 商业人物 → 影视飓风 Tim
│
├── 价值观 / 法理 → 罗翔
│   └── 社会议题 → 地球知识局
│
├── 自然 / 博物 → 学者派
│   └── 工程奇迹 → 星球所 + 影视飓风
│
└── 反常识议题 → 地球知识局
    └── 历史彩蛋 → 御史房
```

## 8. 风格混搭测试

写完方案后，念出来听一遍。如果出现以下情况，混搭失败：

| 现象 | 问题 | 修正 |
|------|------|------|
| 觉得"油腻" | 抒情笔法 + 克制底色不匹配 | 去掉抒情，回到克制 |
| 觉得"突兀" | 钩子风格与正文风格不匹配 | 改钩子 |
| 觉得"散" | 多种笔法混在一起没有主线 | 选定 1-2 种主笔法 |
| 觉得"平" | 风格过单一，没有"主笔法"的辨识度 | 加 1 种 fallback 笔法 |
