# 3 套方案 + 对比矩阵 + 个人推荐模板

> 这是星球研究所撰稿实战 docx 的标准模板。每次生成 3 套方案时，**必须**用这个模板结构输出。

---

## 1. 模板结构（每套方案）

每个方案包含以下 8 个部分：

### 1.1 方案名称
- 一句话概括方案的切入角度
- 例：「千岛之省」/「东吾洋的灯」/「中国最'绿'的省份」

### 1.2 类型与命题
- **类型**：教科书式 / 故事人物式 / 议题反常识式
- **一句话命题**（10-30 字）
- **核心命题（钩子背后的"经典理念"）**

### 1.3 核心骨架
时间码分块表：
| 章节 | 时长 | 命题 |

### 1.4 钩子段落
- 0:00-0:30 的钩子段落（约 200 字）
- 钩子类型标注（数字震撼 / 反常识 / 问题驱动 / 画面驱动 / 故事 / 议题）

### 1.5 三个金句候选
每个方案至少 3 个可作为"截图转发"的金句

### 1.6 完整文案大纲
每个章节的详细文字说明

### 1.7 画面建议
每段镜头语言说明

### 1.8 优劣分析（重要！星球研究所实战 docx 核心）
- **优势**：契合点 / 系列感 / 商业克制度
- **劣势**：可能的学术感 / 拍摄成本 / 商业擦边风险
- **建议**：如何优化

### 1.9 适用主持人
- 桢公子 vs 所长 vs 双人配置 vs 其他

### 1.10 对应 UP 主笔法
- 主笔（哪位 UP 主）
- 辅笔（哪位 UP 主）

## 3. 个人推荐（星球研究所实战 docx 必有部分）

### 3.1 推荐模板

```
## 我的推荐

**如果只能选一套，我推荐方案 X「XXXX」**。原因：

1. **原因 1**（最重要的一条）
2. **原因 2**（与星球研究所风格的契合度）
3. **原因 3**（可执行性 / 商业合规性）
4. **原因 4**（可融合其他方案的元素）

**如果追求 X，选方案 Y**。
**如果预算 / 风格 X，选方案 Z**。

## 接下来你想

1. 选定方案展开成完整 10 分钟脚本正文（约 2500-3500 字）
2. 融合其他方案的关键元素
3. 重做：告诉我你的反馈，我来调整
```

### 3.2 推荐时必须考虑的因素
- 与用户"灵魂四问"和"团队 4 维度"的契合度；
- 商业合规性（如果是商务项目）；
- 星球研究所 / 桢公子风格的契合度；
- 制作可行性（拍摄周期、成本）；
- 长期留存价值（3 年后看仍不过时）；
- 团队资源（导演、动画师、摄影师可调配）。
