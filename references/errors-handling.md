# 异常处理文档（Errors Handling）

> **什么情况下应该拒绝 / 引导 / 降级 / 重做**。
> 本文件定义本 skill 的"边界行为"。

---

## 1. 用户输入问题

### 1.1 没有主题

```
用户说："帮我写个视频脚本"（没指定主题）

回答：
"要写视频脚本，我需要先知道主题。请告诉我：
1. 你想做什么主题？（如'福建微电网'、'故宫600年'）
2. 视频时长？（默认 10 分钟）
3. 风格倾向？（默认星球所，可改）
4. 商务项目还是纯科普？"
```

**不要**：凭想象编一个主题写脚本。

### 1.2 主题超出覆盖范围

参见 [SKILL.md §0.0 不适用场景](../SKILL.md)。

```
用户说："帮我写个 1 分钟的搞笑短视频"

回答：
"1 分钟的纯搞笑短视频不在本 skill 覆盖范围。
本 skill 适用于 ≥3 分钟的地理 / 人文 / 科技 / 工程类深度科普。
建议用通用创作助手或专门的短视频脚本 skill。"
```

### 1.3 主题不明确

```
用户说："写个福建的"

回答：
"'福建'是个大主题，需要进一步明确。请问：
1. 你想写福建的什么？（城市 / 山海 / 美食 / 历史 / 工程）
2. 视频时长？
3. 风格倾向？"
```

### 1.4 用户拒绝回答灵魂四问

```
用户说："别问那么多，直接写"

回答：
"我尽量精简——只问 1 个核心问题：
你想做什么样的视频？给我 1-2 句话描述（包括主题 / 时长 / 风格）。

如果你坚持不给任何信息，我只能基于合理默认写：
- 主题：（从你的话中提取的关键词）
- 时长：10 分钟
- 风格：星球所
- 方案数：3 套

不满意可以随时调整。要这样吗？"
```

## 2. 任务执行问题

### 2.1 创作者研究遇到陌生领域

```
情况：用户要"写一个关于量子计算的科普脚本"，
      但量子计算不在 creator-database.md 已有 10 位 UP 主覆盖范围。

处理：
1. 在 creator-database.md 之外，再找 1-2 位相关 UP 主
   （如"妈咪说" / "中科院物理所" / "李永乐老师"）
2. 风格 fallback 切换到 Fallback 5（学者派）
3. 在评估报告里说明"本方案的 fallback 风格是 XX"
```

### 2.2 事实研究数据不足

```
情况：用户要"写一个关于南极冰盖融化的脚本"，
      但研究后只找到 5 个硬数据（< 10 个要求）。

处理：
1. 扩大事实研究方向（再开 2-3 个 Explore 子代理）
2. 明确告知用户"事实素材不足，建议补充 X / Y / Z 方向"
3. 在方案开头标注"事实待补"——避免输出"无源数据"
```

### 2.3 三连接无法建立

```
情况：用户要求"写个上海和纽约对比"，
      强制三连接时找不到清晰的"物理连接"。

处理：
1. 不强行做三连接——这场景适合 1-2 个连接，不适合 3 个
2. 在评估报告里说明"本方案适用 2 连接而非 3 连接，原因：XX"
3. 保留三连接的"工具价值"——作为"理想标准"而非"硬性规则"
```

### 2.4 5 阶段情绪弧线无法落地

```
情况：用户要"写个 2 分钟的反常识爆款"，
      5 阶段弧线显得过长。

处理：
1. 切到 3-beat 短视频版（钩子 / 紧张 / 回报）
2. 详见 [emotion-arc.md §4 短视频压缩版](./emotion-arc.md)
3. 在评估报告里说明"本方案采用 3-beat 短视频情绪曲线"
```

## 3. 商务项目问题

### 3.1 商务项目客户边界不清晰

```
用户说："我们要和 XX 能源企业合作，写个视频脚本"，
但没说客户期望的"露出度"。

处理：
1. 主动问"客户期望的产品露出度？克制 / 中等 / 直接？"
2. 默认走"克制"模式（人文化包装）
3. 在方案里加"商务项目伦理底线"标注
   （详见 [SKILL.md §8 商务项目伦理底线](../SKILL.md)）
```

### 3.2 客户要求露出过强

```
用户说："我们客户希望产品名在开头就出现 3 次"

处理：
1. 明确告知"这违反'人文化包装'原则，会让视频变成广告"
2. 提供 2 个折中方案：
   a. 开头克制露出 + 中段自然融入
   b. 结尾出现 1 次（不重复）+ 致谢页完整
3. 让用户和客户重新对齐
4. 不接受"广告式脚本"——这违反商务项目伦理底线
```

### 3.3 商务项目数据敏感

```
情况：商务项目涉及"X 省的能源数据"，
      但用户没有数据来源授权。

处理：
1. 明确告知"数据需要客户授权"
2. 暂用"占位数据"（如"超过 60% 的清洁能源占比"）
3. 提醒用户"上线前必须由客户确认数据准确性"
4. 在致谢页标注"数据由 XX 客户授权提供"
```

## 4. 史实 / 数据问题

### 4.1 史实错误

```
情况：撰稿过程中发现自己的初稿里有史实错误。

处理：
1. 立即停止推进
2. 修正 + 加权威来源
3. 在交付文档的"修订记录"里标注"已修正"
4. 重新走质量门 10 项检查
```

### 4.2 数据冲突

```
情况：发现同一个数字有 2 个不同来源给出不同值。

处理：
1. 列出所有来源和数值
2. 优先采用官方 / 学术来源
3. 如必须用某个数值，明确标注"按 XX 来源"
4. 在交付文档里说明"数据存在不同口径，已优先采用 XX"
```

### 4.3 没有数据来源

```
情况：用户提供的数字没有来源。

处理：
1. 拒绝使用——每个数字必须带来源
2. 告知用户"请提供 X 数据的来源"
3. 如确实查不到来源，标注"数据存疑，待核实"
```

## 5. 内容安全问题（一票否决）

| 类别 | 触发条件 | 处理 |
|------|---------|------|
| **政治敏感** | 涉政 / 涉民族 / 涉宗教 / 涉领土 | 整段删除或拒发 |
| **价值观偏离** | 性别歧视 / 地域歧视 / 职业歧视 | 修正或拒发 |
| **伪科学** | 缺乏科学依据的"事实" | 修正或加免责声明 |
| **暴力 / 色情** | 任何形式的暴力 / 色情内容 | 立即停止，拒绝 |
| **未成年人** | 涉及未成年人的不当内容 | 立即停止，拒绝 |
| **个人隐私** | 暴露个人隐私信息 | 删除相关段落 |
| **虚假宣传** | 商务项目的虚假信息 | 拒绝，按事实重写 |

**处理流程**：
1. 立即停止当前任务
2. 明确告知用户触发的安全规则
3. 提供合规替代方案（如"这个角度涉及政治敏感，建议改写为 XX 角度"）
4. 不接受"用户坚持"——内容安全是不可协商的底线

## 6. 技术 / 工具问题

### 6.1 子代理失败

```
情况：Phase 2 事实研究的 Explore 子代理超时或失败。

处理：
1. 重试 1 次（间隔 1 分钟）
2. 仍失败 → 切换到"自研究"模式（不依赖子代理）
3. 明确告知用户"事实研究深度受限，建议手动补充 X / Y / Z 方向"
```

### 6.2 文件 / 模板缺失

```
情况：references/ 里某个文件缺失。

处理：
1. 立即停止当前任务
2. 告知用户"references/ 里 XX 文件缺失，请补齐后重试"
3. 不使用"占位文件"——这是分发质量问题
```

### 6.3 输出超过上下文

```
情况：方案篇幅超出模型上下文限制。

处理：
1. 拆分为多个子任务
2. 第一个任务：输出核心框架（≤2000 字）
3. 第二个任务：输出 3 套方案详情（每套 ≤1500 字）
4. 第三个任务：输出深化方案（≤3000 字）
5. 主理人汇编
```

## 7. 用户体验问题

### 7.1 用户对方案不满意

```
用户说："这个方案不够好，重做"

处理：
1. 问"具体哪里不满意？"（钩子 / 命题 / 节奏 / 人物 / 数据）
2. 针对性调整，不要整体重做
3. 如果用户要求"换方向"，重新走 Phase 3-4（不要重走 Phase 1-2）
4. 最多 2 轮调整——3 轮以上考虑切换到完整模式
```

### 7.2 用户提了新需求

```
情况：方案写到一半，用户加了新需求（"再加一段关于 XX"）

处理：
1. 评估"加段"对结构的影响
2. 如果影响 5 阶段情绪弧线 → 调整情绪曲线
3. 如果只影响信息密度 → 直接插入（标"补充段落"）
4. 如果用户频繁改需求 → 提醒"建议先明确需求再开始 Phase 4"
```

### 7.3 用户要求"多合一"

```
用户说："3 套方案能不能合在一起？"

处理：
1. 拒绝"机械合"——3 套方案的差异会消失
2. 提供"融合方案"——选定 1 套为主，吸收其他 2 套的 1-2 个亮点
3. 在评估报告里说明"本方案 = 方案 A 主体 + 方案 B 的人物 + 方案 C 的钩子"
```
