# 评估模式（Evaluation Mode）

> **用户已有方案 / 脚本，要打分 / 优化** —— 反向使用本 skill。

---

## 1. 触发条件

用户说以下任一话术，进入评估模式：

- "帮我评估这个方案"
- "看看这个脚本怎么样"
- "我写了初稿，帮我打分"
- "这个有什么问题"
- "优化一下这个方案"
- "按 10 分制打分"
- "符合星球所风格吗"
- 直接贴一段脚本说"帮我看看"

---

## 2. 评估模式的 3 个工作流

### 2.1 工作流 A：纯评估（只打分，不改）

适用：用户只想知道方案质量

```
用户贴方案
       ↓
[1] 通读 1 遍，抓核心命题
       ↓
[2] 按"量化质量门" 10 项打分（每项 0-10 分）
       ↓
[3] 检查"一票否决项" 10 条
       ↓
[4] 输出评估报告
       ↓
[5] 询问是否需要优化
```

### 2.2 工作流 B：评估 + 优化建议（推荐）

适用：用户希望提升方案

```
用户贴方案
       ↓
[1] 同工作流 A
       ↓
[2] 列出"必须改"（一票否决项）
       ↓
[3] 列出"建议改"（质量门不达标项）
       ↓
[4] 列出"可保留"（达标项）
       ↓
[5] 输出优化版（差异化重写，不是简单修改）
       ↓
[6] 询问是否需要进入完整模式重新做
```

### 2.3 工作流 C：诊断 + 重做

适用：方案问题太多，与其改不如重做

```
用户贴方案
       ↓
[1] 同工作流 A
       ↓
[2] 总分 < 60 → 建议重做
       ↓
[3] 按完整模式走 6 阶段
       ↓
[4] 交付新版本
```

## 4. 10 维度评分细则

### 4.1 钩子质量（10 分）

| 分数 | 标准 |
|------|------|
| 9-10 | 从 30 个公式中精准选 1，0-30 秒抓住观众 |
| 7-8 | 公式正确但展开一般 |
| 5-6 | 公式陈旧或没标注 |
| 3-4 | 老套开场（"今天我们来聊聊"） |
| 0-2 | 与正文无关 / 鸡汤开场 |

### 4.2 因果链（10 分）

| 分数 | 标准 |
|------|------|
| 9-10 | 每道递进 ≥3 个因果关键词，层层推进 |
| 7-8 | 因果链清晰但密度低 |
| 5-6 | 部分递进是并列而非因果 |
| 3-4 | 通篇并列，没有递进 |
| 0-2 | 没有结构 |

### 4.3 三连接（10 分）

> 适用于连接差异元素的方案

| 分数 | 标准 |
|------|------|
| 9-10 | 3 层次全满 + 真实人物故事 |
| 7-8 | 3 层次全满但人物薄 |
| 5-6 | 只有 2 个层次 |
| 3-4 | 只有技术连接 |
| 0-2 | 强行类比 / 虚假并列 |

### 4.4 金句密度（10 分）

| 分数 | 标准 |
|------|------|
| 9-10 | 每 2 分钟 ≥1 句金句（10 分钟 ≥5 句） |
| 7-8 | ≥3 句 |
| 5-6 | 1-2 句 |
| 3-4 | 0 句但有亮点 |
| 0-2 | 通篇无金句 |

### 4.5 一句话锚定（10 分）

| 分数 | 标准 |
|------|------|
| 9-10 | 20-50 字，含动词 + 情感锚 + 可截图 |
| 7-8 | 长度合适但情感锚弱 |
| 5-6 | 长度过长或过短 |
| 0-2 | 没有锚定句 |

### 4.6 桢公子适配（10 分）

| 分数 | 标准 |
|------|------|
| 9-10 | 长句占比 ≤30%，每分钟 200-250 字，节奏感强 |
| 7-8 | 节奏基本合理 |
| 5-6 | 长句过多 / 过密 |
| 0-2 | 念出来拗口 |

### 4.7 数据来源（10 分）

| 分数 | 标准 |
|------|------|
| 9-10 | ≥80% 数据带权威来源链接 |
| 7-8 | ≥50% 数据有来源 |
| 5-6 | < 50% 有来源 |
| 0-2 | 数据无来源 / 模糊 |

### 4.8 人物出场（10 分）

| 分数 | 标准 |
|------|------|
| 9-10 | ≥3 个真实人物，姓名 + 场景 + 行动 |
| 7-8 | ≥2 个真实人物 |
| 5-6 | 1 个真实人物或仅群体 |
| 0-2 | 无人物 |

### 4.9 情绪曲线（10 分）

| 分数 | 标准 |
|------|------|
| 9-10 | 2 高 + 1 低 + 结尾缓慢下降 |
| 7-8 | 曲线合理但有 1-2 处平直 |
| 5-6 | 起伏不明显 |
| 0-2 | 曲线平 / 一直高潮 / 一直下坠 |

### 4.10 回环升华（10 分）

| 分数 | 标准 |
|------|------|
| 9-10 | 结尾呼应开头画面 / 数字 / 命题，诗意短句 |
| 7-8 | 有回环但升华一般 |
| 5-6 | 升华但无回环 |
| 0-2 | 结尾突兀 |

## 5. 评估话术模板

### 5.1 邀请用户提供方案

```
为了给你准确评估，请提供：
1. 方案 / 脚本全文（Markdown 优先，纯文本可）
2. 视频时长 + 目标平台
3. 商务项目 / 纯科普？
4. 你特别想我关注哪个方面？（可选）
```

### 5.2 输出评估后的下一步

```
如果评分 ≥ 80 分：
"这个方案质量很高，亮点在 X / Y / Z。剩下的小问题你自己修一下即可。"

如果评分 60-80：
"方案基本可用，但 P0 必改项必须处理（X / Y / Z）。
要不要我按工作流 B 输出优化版？"

如果评分 < 60：
"这个问题较多，建议切换到完整模式重做。
完整模式会走完 6 阶段（创作者研究 → 事实研究 → 3 套方案 → 深化 → 交付），
给你生产就绪的脚本。只需要说'按完整模式重做'就行。"
```
